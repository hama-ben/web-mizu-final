# تصحيح: تحسين بروز وتناسق صورتي الفريق بقسم "جاهز لتجربة Mizu؟"

**المشاكل الحالية المطلوب حلها:**

## 1. الدور/الاسم غير ظاهر إطلاقاً بدون Hover
حالياً الاسم والدور يظهران فقط عند تمرير الماوس (Tooltip)، وهذا يعني 99% من الزوار (خصوصاً بالموبايل حيث لا يوجد Hover أصلاً) لن يعرفوا من هما هذان الشخصان.

**الحل**: اعرض الاسم بشكل دائم ومرئي أسفل كل صورة مباشرة (بدون الحاجة لأي تفاعل)، بخط صغير أنيق:
```
[صورة زراري محمد]
   زراري محمد
  مطور ومصمم
```
```
[صورة محمد بن علاهم]
  محمد بن علاهم
 مسؤول الدعم
```
- الاسم بخط Bold صغير (حوالي 11-12px)، والدور تحته بخط أخف ولون رمادي فاتح (9-10px)
- حافظ على الـ Tooltip بالـ Hover كتحسين إضافي بالديسكتوب فقط، لكن لا تعتمد عليه كوسيلة وحيدة لعرض المعلومة

## 2. الحجم صغير جداً وسط القسم
الصورتان تائهتان بصرياً وسط قسم كبير وعنوان بارز.

**الحل**: كبّر الصورة والبطاقة كاملة (صورة + اسم + دور تحتها) لتصبح وحدة بصرية أوضح:
- قطر الصورة: من 70-90px الحالي إلى **100-110px**
- تأكد النص (الاسم والدور) الجديد تحتها لا يبدو مزدحماً أو مقروناً بخط رفيع جداً — امنحه وزناً بصرياً يتناسب مع حجم الصورة الأكبر

## 3. تحسين التناسق العام (Alignment & Balance)
**الحل**:
- تأكد المسافة بين كل بطاقة (صورة+اسم+دور) وحافة العنوان **متساوية تماماً** على الجهتين (قِس المسافة بالبكسل، لا تعتمد على تقدير بصري فقط)
- ضع كل بطاقة (الصورة + النص تحتها) داخل حاوية واحدة (Container) بمحاذاة عمودية موحدة (Vertically centered) بحيث تتوسط ارتفاع القسم بالكامل، وليس فقط محاذية لسطر العنوان الأول
- أضف خلفية خفيفة جداً خلف كل بطاقة (مثلاً دائرة كبيرة شفافة جداً بلون Mizu الأزرق بشفافية 5-8%) تمنحها "حضوراً" أوضح وسط الخلفية الداكنة، بدل ما تكون الصورة وحدها معلقة بالفراغ
- تأكد التوهج (Glow) حول حدود الصورتين متطابق تماماً بالشدة واللون بين الاثنين (نفس قيمة `box-shadow` أو `filter: drop-shadow` بالضبط)

## 4. تأكيد إضافي
راجع صورة "زراري محمد" تحديداً — تأكد إنها نفس الصورة الشخصية المرفقة أصلاً (رسمة الشخصية بالستايل الأنمي) وليست صورة مختلفة انحطت بالخطأ مكانها.

## 5. حركة "طفو" خفيفة ومستمرة (Floating Animation)
أضف لكل بطاقة (الصورة بالكامل مع التوهج حولها) حركة عائمة خفيفة ومستمرة طول الوقت، تعطي إحساساً بأنها "حية" لا ثابتة جامدة:

- حركة رأسية بسيطة (Translate Y) بمسافة صغيرة جداً (6-10px لأعلى ولأسفل)
- مدة الدورة الكاملة (Up → Down → Up): حوالي 3-4 ثوانٍ
- استخدم `ease-in-out` (منحنى ناعم بلا توقفات مفاجئة) وليس حركة خطية جامدة
- **مهم**: اجعل الصورتين تتحركان بتوقيت مختلف قليلاً عن بعضهما (Animation delay بحوالي 0.5-1 ثانية فرق بينهما)، حتى لا تطفو الصورتان بشكل متطابق ومتزامن تماماً (يبدو أكثر طبيعية وعشوائية خفيفة، كأنهما يتحركان بشكل مستقل)
- تأكد الحركة خفيفة جداً وهادئة — الهدف "نبض حياة" بسيط، وليس حركة ملفتة تشتت الانتباه عن زر "حمّل التطبيق الآن"

مثال CSS تقريبي للفكرة (عدّله حسب بنية الكود الفعلية):
```css
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-8px); }
}
.team-avatar-1 { animation: float 3.5s ease-in-out infinite; }
.team-avatar-2 { animation: float 3.5s ease-in-out infinite; animation-delay: 0.7s; }
```

---

**الهدف النهائي**: أي زائر يشوف القسم لأول مرة (بدون أي تفاعل منه) يعرف فوراً — بمجرد النظر — إن وراء ميزو شخصين حقيقيين باسميهما ودورهما، وتكون البطاقتين متوازنتين تماماً بصرياً مع العنوان بالمنتصف، وواضحتين الحجم بما يكفي لجذب الانتباه دون ما يطغوا على الزر الرئيسي "حمّل التطبيق الآن".
